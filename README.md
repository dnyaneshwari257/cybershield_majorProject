EduLearn Chat Module (CyberShield)
A secure, real-time chat application with built-in toxicity detection. This project is built with a Python (Flask) backend, a Supabase database, and Google's Perspective API for content moderation.

This application provides a complete user management system and a user-to-user chat interface. 
Its key security feature is the real-time moderation of messages in English, Hindi, and Hinglish, with an automatic lockout system to temporarily ban users who send harmful content.


✨ Key Features
> Secure User Authentication: New user registration with automatic, unique username generation (e.g., EDU25001).
Secure password hashing using Werkzeug. Forced password change for new users on their first login.

> Account Lockout Security: Protection against brute-force login attacks. (Not implemented, but planned) Locks user accounts after multiple failed login attempts.

> Real-Time Chat Module:A clean interface for users to select and chat with other registered users. Real-time message sending and receiving using Supabase Realtime.
Message status tracking (sent, delivered, read).

> AI-Powered Content Moderation: Integrates Google's Perspective API to score messages for toxicity before they are sent.

> Multi-language Support: Configured to moderate English, Hindi (hi), and Hinglish (hi-Latn). Chat Lockout: Automatically locks a user out of the chat for 5 minutes if they send a message with a toxicity score above the threshold.


## 🛠️ Tech Stack

* **Backend:** [Flask](https://flask.palletsprojects.com/) (Python)
* **Database:** [Supabase](https://supabase.com/) (PostgreSQL)
* **Frontend:** HTML / Jinja2, CSS, JavaScript
* **Key Feature:** A custom-built backend API (`/api/moderate-text`) that serves as a secure proxy for content moderation.
* **External Dependency:** [Google Perspective API](https://www.perspectiveapi.com/) (for toxicity analysis)
* **Email Notifications:** [Flask-Mail](https://pythonhosted.org/Flask-Mail/)

-- 1. Create the users table
CREATE TABLE public.users (
    id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
    name text,
    email text UNIQUE NOT NULL,
    username text UNIQUE NOT NULL,
    password text NOT NULL,
    phone text,
    course text,
    must_change_password boolean DEFAULT true,
    created_at timestamp with time zone DEFAULT now(),
    lockout_until timestamp with time zone
);

-- 2. Create the messages table
CREATE TABLE public.messages (
    id bigint PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
    sender_id uuid REFERENCES public.users(id),
    receiver_id uuid REFERENCES public.users(id),
    message text,
    "timestamp" timestamp with time zone DEFAULT now(),
    status text DEFAULT 'sent'
);

-- 3. Create the function to get conversations
CREATE OR REPLACE FUNCTION get_conversation_messages(user_a_id uuid, user_b_id uuid)
RETURNS TABLE (
    id bigint, sender_id uuid, receiver_id uuid, message text, "timestamp" timestamptz, status text
) AS $$
BEGIN
    RETURN QUERY
    SELECT m.id, m.sender_id, m.receiver_id, m.message, m.timestamp, m.status
    FROM public.messages m
    WHERE
        (m.sender_id = user_a_id AND m.receiver_id = user_b_id) OR
        (m.sender_id = user_b_id AND m.receiver_id = user_a_id)
    ORDER BY m.timestamp ASC;
END;
$$ LANGUAGE plpgsql;

-- 4. Create the sequence and function for new usernames
CREATE SEQUENCE edu25_seq START 1;
CREATE OR REPLACE FUNCTION next_edu25_val()
RETURNS int AS $$
BEGIN
    RETURN nextval('edu25_seq');
END;
$$ LANGUAGE plpgsql;


### 5. Configure Environment Variables

This project will not run without its environment variables. Create a file named `.env` in the root of your project (this file is ignored by Git and will remain private).

Copy the block below into your new `.env` file and fill in your secret keys.

```.env
# --- Flask Config ---
SECRET_KEY="YOUR_OWN_RANDOM_FLASK_SECRET_KEY"
BASE_URL="http://localhost:5000"

# --- Supabase Config ---
# Get these from your Supabase Project Settings > API
SUPABASE_URL="https://YOUR_PROJECT_ID.supabase.co"
SUPABASE_KEY="YOUR_SERVICE_ROLE_KEY"
